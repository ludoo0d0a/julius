# Julius - Voice AI Assistant

**Repository:** https://github.com/ludoo0d0a/julius

Julius is a **Kotlin Multiplatform (KMP)** application designed to provide a seamless voice assistant experience on **Android** and **Android Auto**. It also includes a **coding harness** to build features via Jules or Claude Code (queue, multi-account limits, PR/CI lifecycle). See [docs/HARNESS.md](docs/HARNESS.md).

## Project Structure

The project is organized into the following modules:

- **`:shared`**: Contains the core business logic, data models, and cross-platform code. This module handles the voice processing interactions and state management, shared between different platform entry points.
- **`:androidApp`**: The Android application entry point. This module includes the UI for the mobile application and the integration with Android Auto services.

## Features

- **Voice Interaction**: Leverages advanced voice processing to understand and respond to user commands.
- **Android Auto Support**: Optimized specifically for the car environment, allowing for hands-free operation and safe driving experiences.
- **Kotlin Multiplatform**: Built with KMP to share logic efficiently and ensure consistency across platforms.

## Extended Tool Actions (OpenAI/Gemini)

When **Extended actions** is enabled (`settings.extendedActionsEnabled`), Julius exposes these tool actions in `OpenAIAgent` and `GeminiAgent`:

- `get_battery_level` -> `GET_BATTERY_LEVEL`
- `get_volume_levels` -> `GET_VOLUME_LEVEL`
- `play_music` -> `PLAY_MUSIC`
- `play_audiobook` -> `PLAY_AUDIOBOOK`

## Getting Started

### Prerequisites

- Android Studio Koala or newer (recommended for KMP support).
- JDK 17 or higher.

### Building the Project

1. Clone the repository.
2. Open the project in Android Studio.
3. Sync Gradle project.
4. Run the `:androidApp` configuration to deploy to a device or emulator.

### Configuration

Create a `local.properties` file in the root directory if it doesn't exist (it is automatically generated by Android Studio) to define your SDK path and any secret API keys. Use the same key names as environment variables (e.g. `OPENAI_KEY`, `GEMINI_KEY`). See [docs/ENV_VARS.md](docs/ENV_VARS.md) for the full list.

#### Firebase (required for Google sign-in)

Firebase Auth will fail with **"API key not valid"** if `google-services.json` is missing or still the placeholder.

- Download `google-services.json` from **Firebase Console → Project settings → Your apps (Android)**.
- Ensure the Android app package name is **`fr.geoking.julius`**.
- Place the file at `androidApp/google-services.json` (or `androidApp/src/<flavor>/google-services.json`).

```properties
sdk.dir=/path/to/your/android/sdk
OPENAI_KEY=your_openai_api_key
GEMINI_KEY=your_gemini_api_key
```
